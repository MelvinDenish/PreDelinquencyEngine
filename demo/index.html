The Challenge

₹1.8 Lakh Crore Problem

India's retail NPAs crossed ₹1.8 lakh crore in FY2024. Traditional collections start after default — we intervene before.

🔴

Traditional Approach

  • Reactive — acts after missed payment
  • Generic "Pay now" threats
  • No context about WHY customer is struggling
  • Damages relationship (40% churn rate)
  • Costly recovery (₹15,000+ per account)
🟢

Our PDI Engine

  • Proactive — detects stress 30-90 days early
  • GenAI personalized, empathetic messages
  • SHAP explains exactly WHY (salary delay, medical…)
  • Preserves relationship (EMI restructuring offers)
  • Saves ₹8,000+ per intervention

End-to-End Flow

How It Works

From raw transaction to empathetic intervention in under 5 seconds

01

Transaction Ingested

Customer makes an ATM withdrawal at 2 AM, 3rd time this week. Transaction hits Kafka topic transactions.

{"customer_id": "CUST-4821", "type": "ATM_WITHDRAWAL", "amount": 8500, "time": "02:14 AM", "location": "Mumbai"}
02

Feature Engineering

Flink computes rolling aggregates in real-time: atm_withdrawals_count_7d jumps to 8 (normal: 2). Spark batch adds 30-day trends.

atm_withdrawals_7d: 8 ↑ discretionary_spend_7d: ₹42K ↑ lending_app_txns_7d: 3 ↑ salary_delay: 0 days
03

Ensemble Scoring

28 features fed to XGBoost + LightGBM (tabular) + LSTM (temporal sequence). Weighted ensemble produces risk score.

User Stories

Who Uses This System?

1

Spins Up Infrastructure

python main.py up — launches Kafka, PostgreSQL, Redis, MLflow, Flink via Docker Compose.

2

Generates Bank Data

python main.py generate-customers 1000 — creates realistic Indian demographic data with RBI-aligned income brackets.

3

Trains ML Pipeline

python main.py train — 8-step pipeline: dataset build → XGBoost → LightGBM → LSTM → ensemble → SHAP → fairness audit → MLflow registry.

4

Monitors in Production

Prometheus metrics at /metrics, Grafana dashboards, and Evidently drift detection.

1

Sarah Has a Medical Emergency

Hospital bill of ₹2.5L hits. She starts making frequent ATM withdrawals and downloads 2 lending apps.

2

Engine Detects Stress

Flink real-time features spike: atm_withdrawals_7d: 8, lending_app_txns_7d: 3. Ensemble score: 0.82 (Critical).

3

GenAI Crafts Empathetic Message

Groq LLM reads SHAP explanation ("medical expenses") and generates: "Hi Sarah, we noticed some high medical expenses. Would you like to explore our flexible EMI options?"

4

Intervention Prevents Default

Sarah accepts the 3-month EMI restructuring. She never misses a payment. Relationship saved.

1

Opens RM Dashboard

David sees his portfolio at localhost:8050 — risk heatmap, KPI metrics, and a prioritized list of flagged customers.

2

Clicks Sarah's Profile

Sees the SHAP waterfall chart: "High ATM withdrawals (+0.18), Lending apps (+0.14), Medical spend (+0.09)". No guesswork.

3

Gets AI Call Script

GenAI generates: "Context: Medical emergency. Goal: Offer 3-month EMI. Opening: 'Hi Sarah, calling to check in about your recent expenses...'"

4

Resolves on First Call

Armed with context and script, David resolves Sarah's case in one 4-minute call instead of 3 follow-ups.

1

Reviews Model in MLflow

Elena opens localhost:5000 — sees XGBoost AUC 0.88, LightGBM 0.87, LSTM 0.82, Ensemble 0.93. All hyperparameters logged.

2

Runs Bias Audit

python main.py audit-fairness — Fairlearn + AIF360 check for disparate impact across age groups, genders, and regions.

3

Monitors Model Drift

Evidently AI detects when feature distributions shift (e.g., post-COVID spending patterns changed). Triggers retraining.

4

Experiments with Improvements

Uses Optuna for hyperparameter tuning. Tries TabNet and Temporal Fusion Transformer as future model upgrades.
